# VimKeys v0.2 smoke test

Run on a real Mac with Safari installed. Grant Input Monitoring +
Accessibility when prompted; deny everything else. CI cannot exercise this
checklist — TCC grants and Safari behavior require a real user session.

V-M2 deliverables: insert-mode auto-detect via AX, find / history /
reload bindings, the help overlay, the manual InsertModeBehavior setting.

## Setup

1. Build Release and copy to `/Applications`:
   `xcodebuild build -scheme VimKeys -project VimKeys.xcodeproj -configuration Release -destination 'platform=macOS'`
2. Launch VimKeys.app. Grant Input Monitoring + Accessibility when System Settings opens.
3. Confirm menu-bar item shows the keycap-with-v glyph (no slash, no error tint).

## V-M1 regression checks (must still pass)

4. Cmd-Tab to Safari, focus the page (not address bar). Press `j`/`k` → scroll. Press `5j` → 15 lines down. Press `gg` then `G` → jump to top, then bottom.
5. Cmd-Tab away from Safari. Press `j`. Letter `j` types into the other app.

## V-M2: insert mode (auto-detect)

6. With Safari frontmost on a page with focus on the body, status menu reads "VimKeys — Normal".
7. Click into Safari's address bar. Status menu now reads "VimKeys — Insert" (or remains Normal if AX trust isn't fully active — see step 18). Menu-bar icon shows a small "I" badge in the lower-left.
8. Type `j`. Letter `j` types into the address bar — VimKeys is in insert mode and stays out of the way.
9. Press `Esc`. Address bar loses focus, status returns to "VimKeys — Normal", icon returns to the plain v glyph.
10. Press `j` again on the page → scroll, confirming we're back in vim mode.

## V-M2: insert mode (manual)

11. Open Settings → General. Switch "Switch into insert mode" to "Manual only (i to enter)".
12. Click into Safari's address bar. Status remains "VimKeys — Normal" (focus changes are ignored). Pressing `j` will type `j` into the address bar AND scroll the page. *(That's correct manual-mode behavior — explicit only.)*
13. Click back on page body. Press `i`. Status reads "VimKeys — Insert".
14. Press `Esc`. Status returns to "VimKeys — Normal".
15. Switch back to "Auto-detect via Accessibility". Confirm the toggle persists across launches by quitting + relaunching VimKeys.

## V-M2: find / history / reload bindings

16. With Safari frontmost on any page:
    - Press `/` → Safari's native find bar opens.
    - Type a query, then close find with `Esc`.
    - Open find again with `/`, type a query, press `Esc`. Press `n` → "find next" via Cmd+G. Press `N` → "find previous" via Cmd+Shift+G.
17. Navigate around (click links, etc.). Press `H` → history back. Press `L` → history forward.
18. Press `r` → reload. Press `R` → hard reload via Cmd+Option+R (Safari's "Reload Page From Origin"). **Requires Develop menu enabled** (Safari → Settings → Advanced → "Show features for web developers"). Without Develop menu, `R` is a no-op rather than triggering a different action. Verify hard reload via Web Inspector → Network tab → see all resources re-fetched (no `(from disk cache)` entries).

## V-M2: help overlay

19. Press `?`. A floating overlay appears with the bindings reference (scroll, find/history, mode, coming-later sections).
20. Press any key (e.g. `j`). Overlay dismisses. Press `?` again to reopen, then press `Esc` to dismiss.
21. Verify the overlay does NOT take focus from Safari (the page underneath stays the active window — overlay is a non-activating panel).

## V-M2: AX-trust degradation

22. In System Settings → Privacy → Accessibility, revoke VimKeys' grant. Toggle Safari to frontmost. Insert mode auto-detect stops working (focusing the address bar doesn't flip the badge), but scroll bindings still work via Input Monitoring. Re-grant Accessibility and confirm insert-mode auto-detect resumes.

## Wrap

23. Quit VimKeys via the menu-bar dropdown.
