# VimKeys v0.4 smoke test

V-M4 deliverables: vomnibar (URL / tabs), yank current URL (`yy`), open
clipboard URL (`p` / `P`), yank link URL (`yf`) via the hint copy-only
path, view source (`gs`) already shipped in v0.3.

**First-run TCC:** Apple Events permission is requested the first time
`yy` / `o` / `O` / `T` / `p` / `P` is invoked. The OS shows the
"VimKeys would like to control Safari" prompt with VimKeys' usage
string. Grant it; verify under System Settings → Privacy & Security →
Automation → VimKeys → Safari (toggle is on).

## Setup

1. Build Release, install to `/Applications`. Existing TCC grants from
   v0.3 carry over (same Developer-ID signature).
2. Launch VimKeys. Cmd-Tab to Safari with at least one window open.

## V-M3 regression checks

3. `f` → hints, type label → click. `F` → hints, type label → opens in
   new tab. `gi` → focuses search box on duckduckgo.com.

## V-M4: yank current URL (`yy`)

4. On any Safari page, press `y` then `y`. TCC prompt appears the first
   time — grant. Subsequent `yy` calls execute silently.
5. Paste into a text field anywhere; the current Safari URL appears.
6. If you deny the TCC prompt: the menu-bar status flashes a "Grant
   Apple Events access" message via `lastError`. Re-enable via System
   Settings → Privacy & Security → Automation.

## V-M4: vomnibar (`o`, `O`)

7. Press `o` on a Safari page. Vomnibar panel appears centered near
   the top of the screen with placeholder "Type a URL or search".
8. Type `apple.com`, press Enter. Safari navigates to `https://apple.com`
   in the current tab.
9. Press `o` again. Type `claude code`. Suggestion list shows a
   "Search DuckDuckGo" row. Press Enter → DuckDuckGo opens for that
   query.
10. Press `O` (shift-o). Same UI; press Enter on a URL → opens in a
    *new tab* this time.
11. Press `o`, then Esc. Vomnibar dismisses; status returns to "Normal".

## V-M4: tab vomnibar (`T`)

12. Open 3+ tabs in Safari (e.g. news.ycombinator.com, github.com,
    apple.com).
13. Press `T`. Vomnibar appears in "Switch to tab" mode listing all
    open tabs across all windows.
14. Type a partial title (e.g. `news` → narrows to Hacker News). Press
    Enter → Safari jumps to that tab (and brings its window front).
15. Press `T`, use Up/Down arrows to navigate the suggestions, press
    Enter on a different tab.

## V-M4: paste-and-go (`p`, `P`)

16. Copy a URL from anywhere (e.g. `https://example.com`). Press `p`
    in Safari → current tab navigates to that URL.
17. Copy a search-style string (e.g. "swift 6 concurrency"). Press
    `P` → opens in a new tab as a DuckDuckGo search.

## V-M4: yank link URL (`yf`)

18. On a link-heavy page, press `y` then `f`. Hint overlay appears
    (with the same yellow/orange labels as `f`).
19. Type the label for any link. Instead of clicking, VimKeys copies
    its `href` to the clipboard. Paste anywhere to verify.

## V-M4: bookmarks deferred

20. Press `b`. Nothing happens (V-M5 binding deferred until full-disk-
    access TCC story is in place). Same for `B`.

## V-M4: AX-trust + Apple-Events independent

21. Revoke VimKeys' Apple Events (Automation → Safari) toggle.
22. Press `yy`. Nothing copies; status flashes "Grant Apple Events
    access...".
23. Press `f` on a page. Hint overlay still works (uses Accessibility,
    not Apple Events).
24. Re-grant Automation. Press `yy` again; works.

## Wrap

25. Quit VimKeys via the menu-bar dropdown.
