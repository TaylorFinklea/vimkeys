# VimKeys v0.5 smoke test

V-M5 deliverables: per-site disable rules with auto-detection of the
current Safari host. Apple Events polling drives the host detection,
so Apple Events permission must be granted.

**Deferred to a later release:** suspend chord (Esc-Esc), per-tab
suspend tracking, layout-aware keyboard resolution. Status menu has
been updated to read "Off (this site)" when disabled-by-site.

## Setup

1. Build Release, install to `/Applications`. TCC grants carry over.
2. Launch VimKeys. Confirm Apple Events grant is still active (from
   v0.4 smoke test).
3. Open Safari with at least one tab.

## V-M4 regression checks

4. `yy` copies current URL. `o` opens a URL/search in current tab. `T`
   switches tabs. `f` shows hints. `gi` focuses search field on
   duckduckgo.com.

## V-M5: add a host via Settings

5. Open VimKeys Settings → Sites tab. Field is empty by default.
6. Type `gmail.com`, press Enter (or click Add). Entry appears in the
   list below.
7. Navigate Safari to `https://gmail.com` (or its login page).
8. Within ~1.5s (the AppleScript URL poll cadence), the menu-bar
   status flips to "Off (this site)".
9. Press `j` on the gmail.com page. The letter `j` types into the
   page; no scroll. VimKeys is genuinely off.
10. Press `f`. Nothing happens (hint mode is gated on `.normal`).
11. Navigate Safari away (e.g. to news.ycombinator.com). Within ~1.5s,
    status returns to "Normal". `j` scrolls again.

## V-M5: subdomain match

12. Add `ycombinator.com` to disabled list. Navigate to
    `https://news.ycombinator.com/`. Status flips to "Off (this site)"
    (suffix match: `news.ycombinator.com` ends in `.ycombinator.com`).
13. Verify `notycombinator.com` (if reachable) does NOT match — only
    suffix-at-host boundary qualifies.

## V-M5: one-click "Disable current site"

14. Navigate to a fresh page (e.g. `https://example.com/`).
15. Open Settings → Sites → click "Disable current site". `example.com`
    appears in the list, status immediately flips to "Off (this site)"
    on next poll.

## V-M5: remove a host

16. In the Sites list, click the red minus icon next to one entry.
17. Navigate Safari back to that host. Status returns to "Normal"
    within ~1.5s.

## V-M5: persistence

18. Quit VimKeys. Relaunch. Settings → Sites still shows the same
    entries (UserDefaults-backed).

## V-M5: Apple-Events degradation

19. Revoke VimKeys' Automation → Safari grant.
20. Navigate Safari to a known-disabled host. Status stays "Normal"
    (we can't read the URL → no host comparison → no disable). The
    user-facing message: "Grant Apple Events access" flashes via the
    permissions banner. Re-grant; behavior resumes after the next URL
    transition.

## Wrap

21. Quit VimKeys via the menu-bar dropdown.
