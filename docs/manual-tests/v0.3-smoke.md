# VimKeys v0.3 smoke test

V-M3 deliverables: link hints (`f`, `F`), focus-first-input (`gi`), view
source (`gs`), Settings → Bindings tab with hint-alphabet field.

Run on a real Mac with Safari installed. AX permission must be granted —
without it, hint mode no-ops silently. Develop menu must be enabled for
`R` and `gs` to round-trip (Safari → Settings → Advanced → "Show features
for web developers").

## Setup

1. Build Release and copy to `/Applications`. Confirm TCC grants from
   v0.2.3 carry over without prompts (same Developer-ID signature).
2. Launch VimKeys. Menu-bar shows the chevron, status reads "Off" or
   "Normal" depending on Safari frontmost state.

## V-M2 regression checks

3. `j`/`k`/`h`/`l`/`5j`/`gg`/`G` still scroll Safari correctly.
4. Click the address bar → status reads "Insert"; press Esc → back to
   "Normal".

## V-M3: link hints (`f`)

5. Navigate to a link-heavy page (e.g. `https://news.ycombinator.com`,
   any GitHub repo's `/issues`).
6. Press `f`. Yellow/orange/green badges appear next to every clickable
   element. Labels are short single chars or two-char pairs depending on
   target count.
7. Type the label characters for any visible target. Page navigates as
   if you'd clicked it. Status returns to "Normal" automatically.
8. Press `f` again. Type a single non-matching alphabet char (e.g. `z`).
   Hint overlay beeps; nothing happens.
9. Press `f`, then `Esc`. Overlay dismisses; status reads "Normal".

## V-M3: hint, new tab (`F`)

10. Press `F` on the same page. Same overlay appears.
11. Type a label. Safari opens the link in a background tab (Cmd-click
    behavior). Status returns to "Normal".

## V-M3: `gi` (focus first input)

12. Navigate to a page with a search box near the top
    (e.g. `https://duckduckgo.com`).
13. Press `gi`. Focus jumps to the search box; status flips to "Insert"
    (auto-detect catches the AX focus change). The overlay does NOT
    appear — `gi` skips the labels because there's only one expected
    target.
14. Type a query, press Enter; works as a normal text input.

## V-M3: `gs` (view source)

15. On any page, press `gs`. Safari's Web Inspector opens with the page
    source visible. *(Requires Develop menu enabled. Without it, this is
    a no-op.)*

## V-M3: Settings → Bindings tab

16. Open VimKeys Settings → Bindings tab.
17. The "Hint alphabet" field shows `sadfjkl;ehiwopvbnm` by default.
18. Change it to `abc`. Click outside the field to commit.
19. On a Safari page with >3 links, press `f`. Labels now use only the
    `abc` characters (so 4+ targets get multi-char labels like `aa`,
    `ab`, `ac`, `ba`).
20. Click "Reset to default". Field flips back to
    `sadfjkl;ehiwopvbnm`. Quit + relaunch VimKeys; setting persists.

## V-M3: AX trust degradation

21. Revoke VimKeys' Accessibility grant in System Settings. Press `f`
    on a Safari page. Nothing happens (silent no-op — scroll bindings
    still work via Input Monitoring). Re-grant Accessibility; press `f`
    again — hint overlay appears.

## V-M3: hint clutter cap

22. Navigate to a page with thousands of links (e.g.
    `https://en.wikipedia.org/wiki/Index_of_articles`). Press `f`.
    Overlay caps at ~999 targets so labels stay readable. Scroll-and-
    hint again if the target you want isn't on screen.

## Wrap

23. Quit VimKeys via the menu-bar dropdown.
